Система предназначена для ведения справочной информации о клиентах банка, их счетах, видах счетов, отделениях, а также для учета операций начисления/списания и автоматического начисления процентов. Intended пользователь - сотрудник банка, который просматривает данные, редактирует справочники, открывает и закрывает счета, оформляет операции и выполняет начисление процентов.





Список страниц: 

P1 Главная/Навигация
P2 Список отделений
P3 Карточка отделения
P4 Форма отделения (создание/редактирование)
P5 Список клиентов
P6 Карточка клиента
P7 Форма клиента (создание/редактирование)
P8 Список счетов
P9 Карточка счета
P10 Форма открытия счета
P11 Форма закрытия счета
P12 Форма операции по счету (начисление/списание)

Содержимое страниц:

P1 Главная/Навигация
Данные: пункты меню (Отделения, Клиенты, Счета, Операции, Виды счетов, Начисление процентов).
Действия: переход в выбранный раздел.

P2 Список отделений
Данные: таблица отделений (Название, Адрес).
Действия: поиск/фильтр; открыть карточку отделения; добавить отделение.

P3 Карточка отделения
Данные: Название, Адрес; список счетов отделения (Номер, Клиент, Вид, Баланс, Статус).
Действия: редактировать отделение; удалить отделение (если допустимо); перейти к счету из списка (если показываешь список счетов).

P4 Форма отделения (создание/редактирование)
Данные: поля ввода Название, Адрес.
Действия: сохранить; отменить/вернуться.

P5 Список клиентов
Данные: таблица клиентов (Тип, ФИО/Наименование).
Действия: поиск; фильтры (тип клиента; по виду счета); открыть карточку клиента; добавить клиента.

P6 Карточка клиента
Данные: Тип, ФИО/Наименование; контакты (адреса, телефоны, email); контактные лица (для организаций); список счетов клиента (Номер, Вид, Баланс, Отделение, Статус).
Действия: редактировать; удалить (если нет открытых счетов); открыть счет клиенту; открыть карточку счета из списка.

P7 Форма клиента (создание/редактирование)
Данные: тип клиента; ФИО/Наименование; набор контактов (адреса/телефоны/email); набор контактных лиц (для ORG).
Действия: сохранить; отменить/вернуться.

P8 Список счетов
Данные: таблица счетов (Номер, Клиент, Вид счета, Отделение, Статус, Баланс).
Действия: поиск; фильтры (вид счета, отделение, статус, клиент); открыть карточку счета; открыть новый счет.

P9 Карточка счета
Данные: Номер; Клиент; Вид счета; Отделение; Статус; Баланс; список операций (Дата/время, Тип операции, Сумма, Комментарий).
Действия: задать период операций; оформить начисление; оформить списание; закрыть счет; перейти к клиенту; перейти к виду счета.

P10 Форма открытия счета
Данные: выбор клиента; выбор вида счета; выбор отделения.
Действия: создать счет; отменить/вернуться.

P11 Форма закрытия счета
Данные: номер счета; текущий баланс; результат проверки условий закрытия; сообщение причины отказа (если есть).
Действия: подтвердить закрытие; отменить/вернуться.

P12 Форма операции по счету (начисление/списание)
Данные: счет; тип операции (начисление/списание); сумма; комментарий; результат проверки ограничений (если отказ).
Действия: провести операцию; отменить/вернуться.

P13 Список операций (журнал)
Данные: список операций (Дата/время, Тип, Сумма, Счет, Клиент, Отделение).
Действия: фильтр по периоду; фильтры (тип, клиент, счет, отделение, вид счета); переход к карточке счета.

P14 Список видов счетов
Данные: таблица видов счетов (Название; ключевые параметры, условно max_credit, interest_rate, interest_interval, interest_method).
Действия: поиск; открыть карточку вида счета.

P15 Карточка вида счета
Данные: все правила вида счета (лимит кредита и правила погашения, доходность, интервал/метод выплаты процентов, ограничения на операции).
Действия: редактировать правила.

P16 Форма вида счета (редактирование)
Данные: редактируемые правила вида счета (все поля).
Действия: сохранить; отменить/вернуться.

P17 Начисление процентов (запуск + отчет)
Данные: параметры запуска (дата/период); отчет выполнения (сколько операций создано, по каким счетам, какие пропуски/ошибки).
Действия: запустить начисление.

Счета:
P8 -> P9 (выбор счета)
P8 -> P10 (открыть счет)
P9 -> P12 (начисление/списание)
P9 -> P11 (закрыть счет)
P11 -> P9 (после закрытия/отмены)
P12 -> P9 (после проведения/отмены)

Виды счетов:
P14 -> P15
P15 -> P16
P16 -> P15

Проценты:
P17 -> P13 (проверка созданных операций)
P17 -> P9 (переход к счету из отчета)





Схема БД:

(я буду использовать PostgreSQL)

Таблица BRANCH (отделения)
Поля:
- branch_id BIGSERIAL, PK
- name VARCHAR(200), NOT NULL
- address VARCHAR(400), NOT NULL
  Связи:
- BRANCH 1:N ACCOUNT (account.branch_id -> branch.branch_id)

Таблица CLIENT (клиенты)
Поля:
- client_id BIGSERIAL, PK
- client_type client_type_enum, NOT NULL (PERSON, ORG)
- display_name VARCHAR(300), NOT NULL (ФИО для PERSON или наименование для ORG)
- created_at TIMESTAMPTZ, NOT NULL
  Связи:
- CLIENT 1:N ACCOUNT (account.client_id -> client.client_id)
- CLIENT 1:N CLIENT_ADDRESS / CLIENT_PHONE / CLIENT_EMAIL
- CLIENT 1:N CONTACT_PERSON (для организаций)

Таблица CLIENT_ADDRESS (адреса клиента, множественные)
Поля:
- address_id BIGSERIAL, PK
- client_id BIGINT, FK -> CLIENT, ON DELETE CASCADE
- address VARCHAR(400), NOT NULL
- address_type address_type_enum, NOT NULL (LEGAL, POSTAL, OTHER)

Таблица CLIENT_PHONE (телефоны клиента, множественные)
Поля:
- phone_id BIGSERIAL, PK
- client_id BIGINT, FK -> CLIENT, ON DELETE CASCADE
- phone VARCHAR(50), NOT NULL
- phone_type phone_type_enum, NOT NULL (MOBILE, WORK, OTHER)

Таблица CLIENT_EMAIL (email клиента, множественные)
Поля:
- email_id BIGSERIAL, PK
- client_id BIGINT, FK -> CLIENT, ON DELETE CASCADE
- email VARCHAR(254), NOT NULL
- email_type email_type_enum, NOT NULL (MAIN, BILLING, OTHER)

Таблица CONTACT_PERSON (контактные лица организаций)
Поля:
- contact_id BIGSERIAL, PK
- client_id BIGINT, FK -> CLIENT, ON DELETE CASCADE
- full_name VARCHAR(200), NOT NULL
- phone VARCHAR(50), NULL
- email VARCHAR(254), NULL
- position VARCHAR(120), NULL
  Пояснение: используется для client_type=ORG.

Таблица ACCOUNT_TYPE (виды счетов и правила)
Поля:
- account_type_id BIGSERIAL, PK
- name VARCHAR(200), NOT NULL
- max_credit NUMERIC(14,2), NOT NULL, >= 0 (максимально допустимый кредит; 0 = запрещён любой)
- credit_repay_rule VARCHAR(400), NOT NULL (текстовое описание/код ограничений на погашение кредита)
- interest_rate NUMERIC(7,4), NOT NULL, >= 0 (ставка доходности, условно годовая)
- interest_interval interest_interval_enum, NOT NULL (DAILY, MONTHLY, QUARTERLY, YEARLY)
- interest_method interest_method_enum, NOT NULL (TO_SAME_ACCOUNT, TO_OTHER_ACCOUNT)
- allow_debit BOOLEAN, NOT NULL (разрешены списания)
- allow_credit BOOLEAN, NOT NULL (разрешены начисления)
- min_credit_amount NUMERIC(14,2), NULL (минимальная сумма начисления)
- max_credit_amount NUMERIC(14,2), NULL (максимальная сумма начисления)
- min_debit_amount NUMERIC(14,2), NULL (минимальная сумма списания)
- max_debit_amount NUMERIC(14,2), NULL (максимальная сумма списания)

Таблица ACCOUNT (счета)
Поля:
- account_id BIGSERIAL, PK
- account_number VARCHAR(40), NOT NULL, UNIQUE
- client_id BIGINT, FK -> CLIENT
- branch_id BIGINT, FK -> BRANCH
- account_type_id BIGINT, FK -> ACCOUNT_TYPE
- status account_status_enum, NOT NULL (OPEN, CLOSED)
- opened_at TIMESTAMPTZ, NOT NULL
- closed_at TIMESTAMPTZ, NULL (заполнено если status=CLOSED)
- balance NUMERIC(14,2), NOT NULL (текущий баланс)
- interest_target_account_id BIGINT, FK -> ACCOUNT, NULL (счет-получатель процентов при interest_method=TO_OTHER_ACCOUNT)

Таблица ACCOUNT_TX (операции по счетам)
Поля:
- tx_id BIGSERIAL, PK
- account_id BIGINT, FK -> ACCOUNT, ON DELETE CASCADE
- tx_time TIMESTAMPTZ, NOT NULL
- tx_type tx_type_enum, NOT NULL (CREDIT, DEBIT, INTEREST)
- amount NUMERIC(14,2), NOT NULL, > 0
- comment VARCHAR(400), NULL

Связи

- BRANCH 1:N ACCOUNT
- CLIENT 1:N ACCOUNT
- ACCOUNT_TYPE 1:N ACCOUNT
- ACCOUNT 1:N ACCOUNT_TX
- CLIENT 1:N CLIENT_ADDRESS/CLIENT_PHONE/CLIENT_EMAIL
- CLIENT 1:N CONTACT_PERSON
- ACCOUNT 0..1 -> 0..1 ACCOUNT (interest_target_account_id, самоссылка)





Определение отображения данных БД на страницы пользовательского интерфейса:

P1 Главная/Навигация
Читает/изменяет: нет.

P2 Список отделений
Читает: BRANCH
Изменяет: нет

P3 Карточка отделения
Читает: BRANCH; ACCOUNT по branch_id; CLIENT через ACCOUNT.client_id.
Изменяет: нет (кроме действий перехода/кнопок)

P4 Форма отделения
Создание/редактирование: BRANCH
Удаление: BRANCH (с учетом наличия связанных ACCOUNT)

P5 Список клиентов
Читает: CLIENT
Фильтры по счетам/видам счетов: join CLIENT -> ACCOUNT -> ACCOUNT_TYPE
Изменяет: нет

P6 Карточка клиента
Читает: CLIENT, CLIENT_ADDRESS, CLIENT_PHONE, CLIENT_EMAIL, CONTACT_PERSON, ACCOUNT (по client_id)
Изменяет: нет

P7 Форма клиента
Создание/редактирование: CLIENT, CLIENT_ADDRESS, CLIENT_PHONE, CLIENT_EMAIL, CONTACT_PERSON
Удаление клиента: CLIENT

P8 Список счетов
Читает: ACCOUNT + CLIENT + BRANCH + ACCOUNT_TYPE
Фильтры: по ACCOUNT_TYPE, BRANCH, CLIENT, статусу ACCOUNT
Изменяет: нет

P9 Карточка счета
Читает: ACCOUNT + CLIENT + BRANCH + ACCOUNT_TYPE; ACCOUNT_TX (по account_id, по периоду)
Изменяет: нет

P10 Форма открытия счета
Читает: CLIENT (если выбор клиента на форме), BRANCH, ACCOUNT_TYPE
Создает: ACCOUNT (status=OPEN, opened_at, balance начальный)

P11 Форма закрытия счета
Читает: ACCOUNT + ACCOUNT_TYPE (для проверки условий)
Изменяет: ACCOUNT.status, ACCOUNT.closed_at

P12 Форма операции по счету (начисление/списание)
Читает: ACCOUNT + ACCOUNT_TYPE (проверки разрешений и лимитов)
Создает: ACCOUNT_TX
Изменяет: ACCOUNT.balance (обновление по операции)

P13 Список операций (журнал)
Читает: ACCOUNT_TX + ACCOUNT + CLIENT + BRANCH + ACCOUNT_TYPE (для колонок/фильтров)
Изменяет: нет

P14 Список видов счетов
Читает: ACCOUNT_TYPE
Изменяет: нет

P15 Карточка вида счета
Читает: ACCOUNT_TYPE
Изменяет: нет

P16 Форма вида счета
Изменяет: ACCOUNT_TYPE

P17 Начисление процентов (запуск + отчет)
Читает: ACCOUNT_TYPE (interest_rate/interval/method), ACCOUNT (список счетов по фильтрам), ACCOUNT_TX (для формирования отчета/проверки по периоду при необходимости)
Создает: ACCOUNT_TX (tx_type=INTEREST) на целевой счет начисления процентов
Изменяет: ACCOUNT.balance (обновление балансов)
